★ 웹 데이터의 비용과 효과는 무엇일까?

소비자로서 ...

  • ✔ 볼 수 있다.
  • ✔ 인쇄할 수 있다.
  • ✔ 로컬에(하드 드라이브나 USB 메모리 스틱에) 저장할 수 있다.
  • ✔ 다른 시스템에 수작업으로 데이터를 입력할 수 있다.

공급자로서 ...

  • ✔ 제공하기 쉽다.
“

오픈 라이센스(PDDL, ODC-by, CC0 등)로 공개된 데이터가 웹 상에 존재하는 것은 매우 멋진 일이다. 하지만, 데이터는 문서에 갇혀 있고, 맞춤형 스크레이퍼의 작성없이 문서에서 데이터를 얻기는 어렵다.

”

★★ 웹 데이터의 비용과 효과는 무엇일까?

소비자로서 ★ 웹 데이터로 할 수 있는 모든 것과 함께 다음을 추가로 할 수 있다.

  • ✔ 데이터 수집, 계산, 시각화 등 독점 소프트웨어를 사용해 데이터를 직접 처리할 수 있다.
  • ✔ 다른 (구조화된) 포맷으로 데이터를 내보낼 수 있다.

공급자로서 ...

  • ✔ 여전히 제공하기 쉽다.
“

훌륭하다! 데이터를 웹 상에서 구조화된(즉, 기계가 읽을 수 있는) 방법으로 접근할 수 있다. 하지만, 데이터는 여전히 문서에 갇혀있다. 문서에서 데이터를 얻기 위해서는 독점 소프트웨어에 의존해야 한다.

”

★★★ 웹 데이터의 비용과 효과는 무엇일까?

소비자로서 ★★ 웹 데이터로 할 수 있는 모든 것과 함께 다음을 추가로 할 수 있다.

  • ✔ 특정 소프트웨어의 기능에 한정되지 않고, 자신이 원하는 방법으로 데이터를 다룰 수 있다.

공금자로서 ...

  • ✔ 독점 포맷으로 데이터를 내보내기 위해서 변환기나 플러그인이 필요할지도 모른다.
  • ✔ 여전히 비교적 제공하기 쉽다.
“

놀랍다! 웹을 통해 데이터에 접근할 수 있을 뿐만 아니라 누구나 쉽게 사용할 수 있다. 그러나, 이는 여전히 웹 상의 데이터이지 웹 안의 데이터가 아니다.

”

★★★★ 웹 데이터의 비용과 효과는 무엇일까?

소비자로서 ★★★ 웹 데이터로 할 수 있는 모든 것과 함께 다음을 추가로 할 수 있다.

  • ✔ 어떤 곳(로컬이나 웹 상에)에서라도 링크할 수 있다.
  • ✔ 북마크할 수 있다.
  • ✔ 데이터의 일부를 재사용할 수 있다.

공급자로서 ...

  • ✔ 일반적으로 데이터 분석을 위해 어느 정도의 시간을 투자해게 된다.
  • ✔ 데이터 항목에 URI를 부여해야 하고, 데이터를 표현하기 위한 방법도 생각해야 한다.
  • ✔ 데이터 항목에 대한 정교한 제어가 가능하고, 접근에 대한 최적화(로드 발란싱, 캐싱 등)도 할 수 있다.
“

대단하다! 이제 웹 안의 데이터이다. (가장 중요한) 데이터 항목들이 URI를 가지며, 웹 상에서 공유될 수 있다. 데이터를 표현하기 위한 가장 자연스러운 방법은 RDF를 사용하는 것이다. 하지만, 필요한 경우 Atom과 같은 다른 포맷도 변환 또는 매핑될 수 있다.

”

★★★★★ 웹 데이터의 비용과 효과는 무엇일까?

소비자로서 ★★★★ 웹 데이터로 할 수 있는 모든 것과 함께 다음을 추가로 할 수 있다.

  • ✔ 데이터를 소비하며 더 많은 (관련) 데이터를 발견할 수 있다.
  • ✔ 데이터 스키마에 대해 직접 배울 수 있다.

공급자로서 ...

  • ✔ 웹 상의 다른 데이터와 링크하기 위해서 자원의 투자가 필요해 진다.
  • ✔ 데이터가 발견될 수 있도록 만든다.
  • ✔ 데이터의 가치를 높인다.
“

환상적이다! 이제 다른 데이터에 연결된 웹 안의 데이터이다. 소비자와 공급자는 모두 네트워크 효과를 통한 이익을 얻는다.

”

더 보기

  • 에드 서머스는 이 사이트의 바탕이 된 별점 5 계획의 멋진 표현을 제공했다.
  • 오픈 데이터를 갖고 있다면, 준수 수준을 표시하기 위해 별점 배지를 사용할 수 있다.

CSV 버그를 알려준 앤디 시본, 별점 4/5 예제에 '데이터 하이라이팅'을 제안해 준 커스틴 포스버그, '무엇' 뿐 아니라 '왜'도 설명하라고 제의한 바실리오스 페리스테라에게 영광을 돌린다. 이 사이트는 EC FP7 지원 활동 LOD-Around-The-Clock (LATC)에 의해 제공된다.
